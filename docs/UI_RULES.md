# Правила построения UI редактора

Документ фиксирует соглашения по канве, виджетам, сетке и отображению в приложении Flprog WebUI Editor.

---

## 1. Канва

- **Пропорции** задаются только соотношением сторон (16:9, 3:4, 4:3, 1:1, 2:3, 3:2, 9:16, 21:9 или свои). Единица пропорции не фиксирована: при смене пресета сохраняется текущий масштаб (большая сторона не меняется), пересчитывается вторая сторона.
- **Размер на экране**: канва вписывается в 100% области окна (с небольшим отступом `margin`), с сохранением пропорций. Масштаб: `scale = min((viewportW - margin) / width, (viewportH - margin) / height)`.
- **Координаты канвы** хранятся в логических единицах (`canvasConfig.width`, `canvasConfig.height`). Отображение: `displayWidth = width * scale`, `displayHeight = height * scale`. Контент рисуется в логических координатах и масштабируется через `Group` с `scaleX={scale}`, `scaleY={scale}`.

---

## 2. Сетка

- **Шаг сетки** в логических координатах: `GRID_SIZE = 10` (линии 0, 10, 20, …).
- **Отрисовка**: линии в логических единицах; цвет зависит от фона:
  - светлый фон (luminance ≥ 0.4): основные линии `#ddd`, остальные `#f0f0f0`;
  - тёмный фон: полупрозрачный белый `rgba(255,255,255,0.14)` и `rgba(255,255,255,0.07)`.
- **Привязка к сетке**:
  - В логическом пространстве (при перетаскивании): `snapToGrid(value) = round(value / GRID_SIZE) * GRID_SIZE` — используется в `dragBoundFunc`.
  - В дизайн-пикселях (при сохранении): `snapDesignToGrid(value, scale)` — такое значение в design pixels, что `value / scale` кратно `GRID_SIZE`. Используется в `onDragEnd` и `onTransformEnd` для x, y, width, height.
- При ресайзе в `boundBoxFunc` только проверяется минимальный размер (20×10 в логических единицах), привязка к сетке выполняется при отпускании в `onTransformEnd`.

---

## 3. Виджеты

- **Хранение**: позиция и размер в «дизайн-пикселях» (визуально постоянный размер при смене пропорций канвы). В сторе: `x`, `y`, `width`, `height` в этих единицах.
- **Отрисовка**: в компоненте виджета координаты и размеры переводятся в логические для Konva: `lx = x * invScale`, `lw = width * invScale` и т.д., где `invScale = 1 / displayScale`. Так при любом масштабе канвы виджет сохраняет один и тот же визуальный размер.
- **Новые виджеты и вставка**: начальные x, y, width, height выравниваются по сетке (кратны 10 в сторе; при вставке позиция тоже привязывается).

---

## 4. Текст внутри виджета

- **Высота области текста**: 85% от высоты виджета, `textHeight = 0.85 * lh`.
- **Вертикальное выравнивание**: текст по центру по высоте, `textY = (lh - textHeight) / 2`.
- **Размер шрифта**: пропорционален высоте текста, `textFontSize = max(8, textHeight * 0.65)`.
- Применяется к тексту кнопки, поля ввода, метки (label) и значения слайдера. Подпись над виджетом (caption) не входит в это правило.

---

## 5. Цвет текста (контраст)

- Цвет шрифта выбирается максимально контрастным к фону: чёрный или белый по относительной яркости (WCAG), см. `contrastColor()` в `contrastColor.ts`.
- Для подписи над виджетом — контраст к цвету канвы; для текста на виджете — к цвету виджета (для switch — к цвету состояния).

---

## 6. Выбор цвета (пикер)

- **Предпросмотр**: во время перетаскивания в пикере цвет обновляется с троттлингом (~80 ms), чтобы не нагружать рендер.
- **Сохранение**: в стор цвет записывается только при отпускании мыши (`onPointerUp`) или потере фокуса (`onBlur`). Контраст текста пересчитывается после этого.
- Для канвы и виджета используются отдельные ref и состояние предпросмотра; при смене выбранного виджета предпросмотр цвета виджета сбрасывается.

---

## 7. Константы и ключевые функции

- `GRID_SIZE = 10`
- `snapToGrid(value)` — привязка в логических координатах.
- `snapDesignToGrid(value, scale)` — привязка в дизайн-пикселях под логическую сетку.
- `contrastColor(hexOrRgb)`, `getLuminance(hexOrRgb)` — в `src/contrastColor.ts`.
- Масштаб канвы и размер viewport — в `App.tsx` (viewport через `useViewportSize`).

При изменении логики канвы, виджетов или сетки эти правила следует учитывать и при необходимости обновлять данный документ.
