# Правила отрисовки UI (без редактирования)

Документ задаёт правила, достаточные для воспроизведения того же вида интерфейса при отрисовке без логики редактирования (превью, экспорт, рантайм).

---

## 1. Канва

- **Размеры** в логических единицах: `width`, `height`. Цвет фона: `color` (HEX).
- **Масштаб отображения**: один общий `scale` для всего содержимого. Размер на экране: `displayWidth = width * scale`, `displayHeight = height * scale`.
- Вся графика рисуется в **логических координатах** (0…width, 0…height), затем применяется общая трансформация масштабирования `scaleX=scale`, `scaleY=scale`. Масштабирование выполняется на ширину экрана

---

## 2. Сетка (опционально)

- не нужна

---

## 3. Система координат виджетов

- У каждого виджета: `x`, `y` — позиция левого верхнего угла в логических единицах канвы; `width`, `height` — размеры в тех же единицах.
- При отрисовке: группа виджета помещается в точку `(x, y)`; внутреннее содержимое рисуется в локальных координатах с шириной `lw = width`, высотой `lh = height`.

---

## 4. Текст: общие метрики

Для текста внутри виджета (кнопка, поле ввода, Label, значение слайдера):

- **Высота области текста**: `textHeight = 0.85 * lh`.
- **Смещение по вертикали**: `textY = (lh - textHeight) / 2`.
- **Размер шрифта**: `textFontSize = max(8, textHeight * 0.65)`.

Подпись над виджетом (caption) не использует эти формулы (см. п. 6).

---

## 5. Цвет текста

- **Подпись над виджетом (caption)**: цвет текста = `contrastColor(цвет_канвы)`.
- **Кнопка, поле ввода, значение слайдера**: цвет текста = `contrastColor(цвет_виджета)`. Для переключателя (switch) «цвет виджета» — это цвет текущего состояния (#22c55e или #e5e7eb).
- **Label (метка)**: цвет текста **строго** берётся из данных виджета: `fill = color` виджета. Не пересчитывается по контрасту и не зависит от фона канвы.

Функции: `contrastColor(hexOrRgb)`, `getLuminance(hexOrRgb)` — в `contrastColor.ts`.

---

## 6. Отрисовка по типам виджетов

### Подпись (caption)

- Если тип не `rect` и `caption` не пустой: текст рисуется **над** виджетом.
- Позиция: `x=0`, `y=-18`, `width=lw`, `height=18`, выравнивание по центру по горизонтали и вертикали.
- Цвет: `contrastColor(цвет_канвы)`, `fontSize=12`.

### LED

- Один круг: `radius = lw/2`, центр `(lw/2, lh/2)`.
- Заливка: при `text === '1'` — `color` виджета, иначе `#333`.
- Обводка: `#111`, `strokeWidth=2`.
- Тень: при включённом состоянии `shadowBlur=15`, `shadowColor=color` виджета; иначе без тени.

### Label

- Один текстовый блок: содержимое `text`, область `(0, textY, lw, textHeight)`.
- Выравнивание: по горизонтали — left, по вертикали — middle.
- Цвет текста: **только** `color` виджета (см. п. 5).
- Шрифт: `textFontSize` по п. 4.

### Button, Input, Switch, Rect (прямоугольник)

- **Прямоугольник** (Rect):
  - Размер: `lw × lh`.
  - Заливка: для switch — при `text === '1'` цвет `#22c55e`, иначе `#e5e7eb`; для остальных — `color` виджета.
  - Обводка: только у input — `#ddd`, `strokeWidth=1`; у остальных без обводки.
  - Скругление: button — 5, input — 2, switch — `lh/2`, rect — 10.

- **Switch дополнительно**: круглый «ползунок» — круг радиусом `lh/2 - 3`, центр по горизонтали при `text === '1'` в `lw - lh/2`, иначе в `lh/2`; по вертикали `lh/2`. Заливка `#ffffff`, лёгкая тень.

- **Текст на кнопке и в поле ввода**: один текстовый блок в области `(0, textY, lw, textHeight)`, выравнивание center/middle, цвет `contrastColor(цвет_виджета)`, размер шрифта по п. 4.

### Slider

- Прямоугольник как у button/input (без отдельной «дорожки»), плюс текст **справа** от виджета: позиция по x = `lw + 8`, область по ширине 50, по высоте `textHeight`; выравнивание left/middle; содержимое — значение `text`; цвет `contrastColor(цвет_виджета)`, размер шрифта по п. 4.

---

## 7. Что не рисуется при «чистой» отрисовке

- Нет выделения (Transformer), нет осей центра канвы, нет контура по краям виджета.
- Нет элементов интерфейса редактирования
- Нет сетки

---

## 8. Константы

- `GRID_SIZE = 5` — для сетки и (при необходимости) привязки координат.
- `contrastColor`, `getLuminance` — из `src/contrastColor.ts`.

Этих правил достаточно для воспроизведения внешнего вида UI при отрисовке без редактирования.
