# Симулятор ESP

Локальный HTTP-сервер, эмулирующий API прошивки FLProg WebServer. Позволяет тестировать UI без реального ESP: просмотрщик подключается к симулятору и получает конфиг/состояние, отправляет команды `/set`.

## Запуск

```bash
npm run simulator
```

Сервер слушает порт **31337** (или `SIMULATOR_PORT` из окружения).

## Использование

1. Запустите симулятор: `npm run simulator`.
2. В редакторе откройте «Generate Code» и в блоке «Симулятор ESP» нажмите **Отправить проект в симулятор**.
3. Подключите просмотрщик к `http://localhost:31337` (или откройте ссылку из редактора).

## API (как на реальном устройстве)

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/load` | Загрузить проект. Тело: `{ "widgets": [...], "canvasConfig": {...} }`. Вызывается из редактора. |
| GET | `/config` | Конфигурация UI (вкладки и виджеты). |
| GET | `/state` или `/state?fmt=short` | Текущее состояние переменных. |
| GET/POST | `/set?var=...&value=...` | Установить переменную (как при действии пользователя в UI). |
| GET | `/ping` | Ответ `ESP32-WEBUI`. |

Переменные и порядок state совпадают с сгенерированным кодом (.ino).
