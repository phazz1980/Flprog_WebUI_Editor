# В каком виде виджет получает и отправляет команды

## 1. Исходящая команда (viewer → устройство/симулятор)

При нажатии кнопки, переключении switch, движении слайдера или вводе в input вьювер отправляет **установку переменной** одним из способов:

### HTTP-запрос

- **Метод:** `GET` (в текущем коде вьювера) или `POST`
- **URL:** `/set` с query-параметрами

Пример для кнопки (нажатие — «вниз»):

```http
GET /set?var=ИМЯ_ПЕРЕМЕННОЙ&value=1
```

Отпускание кнопки:

```http
GET /set?var=ИМЯ_ПЕРЕМЕННОЙ&value=0
```

Параметры:

| Параметр | Описание |
|----------|----------|
| `var`    | Имя переменной. Для кнопки и LED — как в конфиге (`varName`). Для switch/slider/input — имя с суффиксом `_out`, например `relay_out`. |
| `value`  | Строковое значение: `"0"`, `"1"`, число для слайдера, строка для input и т.д. |

В коде вьювера это делается так:

```ts
// viewer/src/App.tsx
await fetch(`${base}/set?var=${encodeURIComponent(varName)}&value=${encodeURIComponent(value)}`);
```

Виджет в момент клика не «получает» команду — он **инициирует** вызов `onSetVar(varName, value)`, откуда и уходит этот `fetch`.

## 2. Что виджет «получает» — состояние от устройства

Виджет не получает отдельные «команды», а получает **общее состояние** всех переменных через опрос:

### Запрос состояния

- **Метод:** `GET`
- **URL:** `/state?fmt=short`

Пример:

```http
GET /state?fmt=short
```

### Ответ (формат `short`)

JSON-объект: ключи — индексы слотов состояния (`"0"`, `"1"`, …) и служебные ключи `"s"` (sound_enabled), `"m"` (ui_message).

Пример:

```json
{
  "0": 0,
  "1": 42,
  "2": 1,
  "3": 0,
  "s": 0,
  "m": ""
}
```

Порядок слотов совпадает с порядком виджетов с переменными в конфиге; у slider/switch/input два слота подряд: `_out`, затем `_in`.

### Как виджет использует состояние

- При разборе `/config` каждому виджету присваивается **stateIndex** (индекс его слота в state).
- При каждом ответе `/state` вьювер кладёт этот JSON в общий state и для каждого виджета вызывает:
  - `rawVal = getStateValueByIndex(state, stateIdx)`
  - `displayText = displayValue(widget, rawVal, widget.text)`
- На канве рисуется уже готовый `displayText` (для кнопки после правки — всегда текст с кнопки, для LED/switch — отображаемое значение 0/1 и т.д.).

То есть виджет «получает» команду в виде **значения в общем state по своему stateIndex**, а не отдельным сообщением.

## 3. Конфиг виджета (откуда берётся varName и stateIndex)

При подключении вьювер запрашивает `GET /config`. Ответ разбирается в `viewer/src/configParser.ts`.

Формат конфига (упрощённо): `[ tabNames[], compactWidgets[], [ width, height, color ] ]`.

Элемент **compactWidgets**: массив виджетов, каждый — массив вида  
`[ id, typeCode, x, y, width, height, color, text, tabIndex, varName? ]`.

Внутри вьювера виджет представлен как **RuntimeWidget** (`viewer/src/types.ts`):

- **varName** — имя переменной (для кнопки оно же уходит в `var` в `/set`).
- **stateIndex** — индекс в массиве/объекте state, по которому виджет «читает» своё значение из ответа `/state?fmt=short`.

## 4. Краткая сводка

| Действие | Кто инициирует | Формат «команды» |
|----------|----------------|-------------------|
| Нажатие кнопки | Вьювер по pointerdown/pointerup | `GET /set?var=<varName>&value=1` или `value=0` |
| Переключатель / слайдер / input | Вьювер при изменении | `GET /set?var=<varName>_out&value=<строка>` |
| Узнать текущее значение | Вьювер по таймеру | `GET /state?fmt=short` → JSON `{ "0": ..., "1": ..., "s": ..., "m": ... }` |

Виджет «получает» команду только опосредованно: как **значение в общем state** по своему **stateIndex** после очередного запроса `/state?fmt=short`.
