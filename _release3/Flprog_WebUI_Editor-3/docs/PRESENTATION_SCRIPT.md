# Сценарий видео-презентации: Flprog Web UI

Ориентировочная длительность: **8–12 минут**. Можно сократить, убрав детали, или растянуть демо.

---

## 1. Вступление (30–60 сек)

**Что сказать:**
«Привет! Сегодня покажу **Flprog Web UI** — веб-инструмент для проектирования интерфейсов под микроконтроллеры с прошивкой Flprog WebServer. Всё в браузере: рисуешь экран, настраиваешь виджеты, получаешь код под Arduino и блок для FLProg — без установки тяжёлых программ.»

**Что показать:** открыть редактор в браузере, общий вид (канва, панели слева/справа).

---

## 2. Редактор: канва и вкладки (1–1,5 мин)

**Что сказать:**
«Интерфейс строится на **канве** с настраиваемыми пропорциями и цветом фона. Есть пресеты: 16:9 для монитора, 4:3 для планшета, 9:16 для смартфона — или свои значения. Можно сделать несколько **вкладок** и раскладывать элементы по экранам.»

**Что показать:**
- Правая панель → блок «Canvas»: ширина, высота, пропорции, цвет фона.
- Смена пресета (16:9 → 9:16) — канва меняет форму.
- Добавление вкладок и переключение между ними.

---

## 3. Виджеты: добавление и расстановка (2–2,5 мин)

**Что сказать:**
«В левой панели — **виджеты**: кнопка, слайдер, поле ввода, LED, метка (Label), прямоугольник и переключатель (Switch). Выбираем тип — виджет появляется на канве. Его можно **перетаскивать** и **менять размер** за углы. Координаты привязываются к сетке. В правой панели задаём точные X, Y, ширину и высоту; поддерживаются проценты от размера канвы.»

**Что показать:**
- Добавление по одному виджету каждого типа (хотя бы кнопка, слайдер, LED, переключатель).
- Перетаскивание, ресайз, смена позиции в панели свойств (X, Y, Ш, В).
- Кратко: подпись над виджетом (caption), цвет виджета в панели свойств.

---

## 4. Переменные и привязка к МК (1 мин)

**Что сказать:**
«Каждый виджет можно связать с переменной на микроконтроллере: в свойствах задаётся **имя переменной**. Для кнопки, слайдера, поля ввода и переключателя значение уходит на устройство; LED и Label показывают данные с устройства. Поддерживаются и кириллические имена.»

**Что показать:**
- Выбор виджета → в панели свойств поле «Variable Name», ввод имени (например `led1`, `brightness`).

---

## 5. Генерация кода: .ino и блок .ubi (1,5–2 мин)

**Что сказать:**
«Когда макет готов, нажимаем **Generate Code**. Редактор выдаёт готовый Arduino-код: конфиг интерфейса, переменные и обработчики. Можно **скачать .ino** и открыть в Arduino IDE или в FLProg. Отдельно есть кнопка **«Получить блок (.ubi)»** — скачивается блок для вставки в проект FLProg без ручного копирования кода.»

**Что показать:**
- Кнопка «Generate Code» → открывается окно с кодом.
- «Download .ino» → файл сохраняется.
- «Получить блок (.ubi)» → скачивается .ubi файл.
- Коротко пролистать сгенерированный код (объявления, конфиг, setup/loop).

---

## 6. Симулятор ESP (1–1,5 мин)

**Что сказать:**
«Проверить интерфейс можно **без железа**. Запускаем симулятор командой `npm run simulator` — поднимается локальный сервер с теми же API, что у прошивки Flprog WebServer. В редакторе жмём «Отправить проект в симулятор» — конфиг уходит на localhost. Дальше открываем **просмотрщик** (viewer), вводим адрес симулятора и смотрим, как будет выглядеть и работать интерфейс на устройстве.»

**Что показать:**
- Терминал: `npm run simulator` (или показать уже запущенный).
- В редакторе: «Отправить проект в симулятор».
- Открыть viewer в другой вкладке → IP localhost, порт 31337 → «Подключиться».
- Коротко: нажатие кнопки, движение слайдера, переключатель — запросы уходят в симулятор.

---

## 7. Просмотрщик (Viewer): работа с реальным устройством (1–1,5 мин)

**Что сказать:**
«**Просмотрщик** — отдельное приложение: вводишь IP и порт устройства с Flprog WebServer, нажимаешь «Подключиться». Загружается конфиг и состояние с МК, интерфейс рисуется по тем же правилам. Кнопки и слайдеры отправляют команды на устройство. Есть поддержка звука: уведомление и тревога; при тревоге появляется кнопка «Выключить тревогу». Просмотрщик можно развернуть на своём сервере или GitHub Pages и заходить с телефона или планшета.»

**Что показать:**
- Viewer: ввод IP (или снова localhost для симулятора), «Подключиться».
- Переключение вкладок, нажатие кнопки, изменение слайдера.
- Упоминание звука и кнопки отключения тревоги (можно без детального показа).

---

## 8. Демо-режим и копирование (30 сек)

**Что сказать:**
«В редакторе есть **демо-режим**: можно включить его и прямо на канве потыкать кнопки, подвигать слайдер, ввести текст в поле — без отправки в симулятор. Виджеты поддерживают **копирование**: Ctrl+C и Ctrl+V, в том числе при русской раскладке.»

**Что показать:**
- Включение демо-режима, пара действий с кнопкой и слайдером.
- Ctrl+C по виджету, Ctrl+V — появилась копия.

---

## 9. Заключение (30–60 сек)

**Что сказать:**
«Итого: **Flprog Web UI** — это редактор интерфейсов в браузере, генерация .ino и .ubi для FLProg, симулятор для проверки и отдельный просмотрщик для работы с реальным устройством. Всё открыто, можно собирать и деплоить у себя. Ссылки на репозиторий и инструкции — в описании. Спасибо за внимание!»

**Что показать:** финальный слайд или страница README с ссылками (репозиторий, как запустить, как задеплоить viewer).

---

## Подсказки для съёмки

| Момент | Совет |
|--------|--------|
| Редактор | Заранее откройте проект с 2–3 вкладками и разными виджетами — не тратьте время на создание с нуля в кадре. |
| Симулятор | Запустите `npm run simulator` до начала записи; покажите только отправку проекта и подключение viewer. |
| Панели на мобильном | Если показываете мобильную вёрстку: панели сворачиваются, открываются по кнопке ☰. |
| Генерация кода | Имя проекта при скачивании .ino берётся из поля «Project name» в панели — заполните его заранее. |

---

## Вариант короткой версии (3–4 мин)

1. Вступление + общий вид редактора (30 сек).
2. Канва + один пресет, 2–3 виджета (кнопка, слайдер, LED), перетаскивание и ресайз (1 мин).
3. Generate Code → Download .ino и «Получить блок .ubi» (40 сек).
4. Симулятор + viewer: отправка проекта, подключение, пара действий (1 мин).
5. Заключение (30 сек).

Удачи со съёмкой.
