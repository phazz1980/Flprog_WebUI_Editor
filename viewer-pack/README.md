# Набор файлов для просмотрщика UI

Эта папка содержит всё необходимое для переноса в проект **просмотрщика UI** (рантайм-клиент), который получает конфиг и состояние с МК и рисует интерфейс по правилам Flprog WebUI.

## Содержимое

| Файл | Назначение |
|------|------------|
| **contrastColor.ts** | Функции `contrastColor(hexOrRgb)` и `getLuminance(hexOrRgb)` для цвета текста и теней (п. 5, 6 правил). Обязателен для переноса или точной переработки на другом языке. |
| **viewerConstants.ts** | Константа `GRID_SIZE = 5` (п. 8 правил). |
| **UI_RENDER_RULES.md** | Спецификация: канва, координаты, метрики текста, отрисовка по типам виджетов, форматы `/config` и `/state`, сопоставление state с виджетами, display value, вкладки, порядок отрисовки. |
| **README.md** | Этот файл. |

## Как использовать

1. Скопируйте папку `viewer-pack` в проект просмотрщика (или отдельно `contrastColor.ts`, `viewerConstants.ts` и `UI_RENDER_RULES.md`).
2. Подключите `contrastColor` и при необходимости `viewerConstants` в коде отрисовки.
3. Реализуйте в просмотрщике логику по **UI_RENDER_RULES.md**:
   - загрузка GET `/config`, разбор `[tabIds, compactWidgets]`;
   - загрузка GET `/state` (или `/state?fmt=short`), построение списка виджетов с переменными и сопоставление с state (п. 9.4);
   - вычисление display value по п. 9.5;
   - отрисовка канвы и виджетов по п. 1–6 в порядке п. 9.7;
   - при необходимости — вызов GET/POST `/set?var=...&value=...` при действиях пользователя.

## Что не входит в набор

- Код редактора (выделение, панели свойств, генерация кода).
- Сетка, Transformer, элементы редактирования — при «чистой» отрисовке не рисуются (п. 7).
- `PIXELS_PER_UNIT`, `snapToGrid` — нужны только в редакторе; в просмотрщике масштаб задаётся по п. 1 (общий `scale`).

## Источник

Файлы подготовлены из репозитория **Flprog_WebUI_Editor** (редактор интерфейса для МК с Flprog WebServer). Конфиг и state отдаёт прошивка, сгенерированная этим редактором.
